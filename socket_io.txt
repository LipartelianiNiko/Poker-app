Looking at your code, I can spot several bugs. Let me go through them:

**1. `advanceturn()` is called inside action methods AND then `checkandadvance()` is called again in `handleaction()` — double advancing**

In `handleaction()` you call `this.checkandadvance()` after the switch, but each action method (`docall`, `doraise`, etc.) already calls `this.advanceturn()` which itself calls `this.checkandadvance()`. So `checkandadvance` runs twice every action. Remove the `this.checkandadvance()` call at the bottom of `handleaction()`.

**2. `advanceturn()` calls `checkandadvance()` but `checkandadvance()` calls `advanceStage()` which resets state — then `advanceturn()` still continues after returning from `checkandadvance()`**

The flow is tangled. `advanceturn` should only move the turn pointer. Stage advancement should be a separate concern triggered only from `handleaction`. The `checkandadvance()` call inside `advanceturn()` causes cascading recursive calls.

**3. `resetHasActed()` also resets `contribution` to 0 — but it's called mid-hand during raise**

```js
resetHasActed(){
    for(let p of this.table.players){
        if(!p.folded && !p.isAllin){
            p.hasActed=false
            p.contribution=0  // ← BUG: wipes contributions mid-street
        }
    }
}
```

This is called from `doallin()` when an all-in causes a raise. Resetting `contribution` mid-street breaks your `tocall` calculation for everyone else. Contributions should only reset between streets, not on a raise. You need two separate methods — one that only resets `hasActed` (for raises), and one that resets both (for new streets).

**4. `advanceStage()` calls `checkandadvance()` at the end of showdown, which immediately calls `advanceStage()` again into "waiting" then into a new game — without any pause or client notification**

```js
case "showdown":
    this.handevaluation()
    this.giverevards()
    this.checkandadvance() // ← triggers advanceStage again immediately
```

You probably want a deliberate `advanceStage()` call here, not `checkandadvance()`.

**5. `dealturn()` is never called in `advanceStage()`**

```js
case "turn":
    break  // ← missing this.dealturn()
```

**6. `highestbet` initialized referencing `this.bigBlind` before it's defined**

In the table constructor:
```js
this.highestbet = this.bigBlind;  // ← bigBlind not defined yet at this point
```
`this.bigBlind` is defined further down. This resolves to `undefined`. Move `smallBlind` and `bigBlind` definitions to the top of the constructor.

**7. In `advanceturn()`, the loop logic can get stuck or land on an already-acted player**

```js
for(let i=0; i<n; i++){
    this.table.currentTurn = (this.table.currentTurn + 1) % n;
    const p = this.table.players[this.table.currentTurn]
    if(!p.hasActed && !p.isAllin && !p.folded){
        break;
    }
}
```

If no valid player is found (everyone acted/folded/allin), the loop ends without breaking and `currentTurn` lands on whoever it stopped at last — which could be an invalid player. This should fall through to `checkandadvance()` instead.

3. Split `resetHasActed()` into `resetActedOnly()` (for raises) and `resetForNewStreet()` (for stage changes)
4. Add `this.dealturn()` in the `"turn"` case of `advanceStage()`
5. Move `smallBlind`/`bigBlind` to top of table constructor
6. Fix the showdown `checkandadvance()` → explicit `advanceStage()`
